# 德胧酒店Skill批量生成与发布平台 - 整体设计方案

> **项目目标**：为德胧集团每一家酒店生成专属AI Skill并发布到全球主流Skill市场
> **负责人**：德胧AI龙虾军团
> **创建日期**：2026-04-17

---

## 一、项目背景与愿景

### 1.1 行业标杆参考

**金谷园饺子馆案例**：
- 北京北邮旁边的一家饺子馆火了
- 老板（北邮计算机专业毕业）利用AI编程工具，仅用数小时完成开发
- 基于MCP协议（Model Context Protocol）
- Skill调用腾讯云MCP接口，支持美团帐号授权在线取号
- 功能涵盖：餐厅信息、外卖服务、生饺子打包、Wi-Fi查询、排队等6项核心场景
- 上线后冲上热搜，成为"全国第一家接入AI智能体生态的传统餐饮品牌"
- GitHub开源地址：https://github.com/JinGuYuan/jinguyuan-dumpling-skill

**关键洞察**：
1. Skill = 面向AI的"业务接口"，让AI能够理解业务
2. SKILL.md是核心，包含元数据+Agent指令
3. 可对接MCP实现实时数据（排队、预订等）
4. 开源发布后引发大量传播，节省大量营销费用

### 1.2 德胧的机会

德胧集团拥有：
- **40+**开元品牌酒店（开元名都、开元名庭、曼居等）
- **大量**优质商旅客户资源
- **完善**的会员体系（开元商祺会）
- **丰富**的宴会、会议资源

**如果每家酒店都有一个Skill**：
- 顾客可以通过任意AI助手查询酒店信息
- 潜在客群覆盖所有AI用户（龙虾用户）
- 品牌曝光度指数级提升
- 开创酒店行业"AI原生服务"新标准

## 二、Skill核心结构

```

├── package.json          # 【可选】依赖管理
├── CLAWHUB.md           # 【可选】发布说明
├── src/
│   ├── index.ts         # 【可选】MCP服务入口
│   └── tools/           # 【可选】工具函数
└── .github/
    └── workflows/       # 【可选】CI/CD自动发布
```

### 2.2 SKILL.md核心要素

```yaml
---
name: delonix-tianjin-riwan        # 唯一标识
description: 天津瑞湾开元名都...    # 简短描述

  - "天津瑞湾"
  - "瑞湾开元名都"
version: "1.0.0"                   # 语义化版本
brand: "德胧集团 · 开元品牌"         # 品牌标识
---

# 酒店名称 · SKILL

## 基础信息
- 地址：...
- 电话：...
- 营业时间：...

## 核心功能
1. **客房查询**
2. **预订咨询**
3. **会议宴会**
4. **会员权益**
...

## 对话示例
- "天津瑞湾地址在哪？"
```

## 五、Skill生成方法论

### 5.1 三步法

```
Step 1: 定义触发词（triggers）
        ↓
        用户怎么称呼这个酒店/服务
        示例：天津瑞湾 | 瑞湾开元名都 | 天津酒店

Step 2: 整理知识库（knowledge）
        ↓
        地址、电话、营业时间、特色服务
        示例：
        - 基础信息（地址/电话/星级）
        - 客房类型与价格
        - 餐饮服务
        - 会议宴会设施
        - 会员权益

Step 3: 设计交互场景（scenarios）
        ↓
        用户会问什么？怎么回答？
        示例：
        Q: 天津瑞湾地址在哪？
        A: 酒店位于天津市滨海新区...
        
        Q: 海景房多少钱？
        A: 海景房价格随季节浮动...
```

### 5.2 指挥AI搭建Skill的指令模板

```markdown
你是一个Skill架构师，请帮我为[酒店名称]创建一个OpenClaw Skill：

## 酒店基础信息
- 名称：[填写]
- 地址：[填写]
- 电话：[填写]
- 星级：[填写]
- 品牌：[填写]

## 核心服务
1. [填写]
2. [填写]
3. [填写]

## 特色亮点
- [填写]
- [填写]

## 目标用户
- [填写]

## 预期交互场景
- "用户会问..."
- "AI应该回答..."

请生成完整的SKILL.md文件，并提供GitHub/ClawHub发布命令。
```

## 八、风险与注意事项

### 8.1 授权合规
- 发布前需获得酒店运营方授权
- 敏感信息（内部系统接口）不得暴露
- 价格信息建议标注"以官网为准"

### 8.2 信息准确性
- 生成的Skill内容需人工审核
- 建议设置季度更新机制
- 联系方式变更需及时同步

### 8.3 安全扫描
- 安装外部Skill前建议使用skill-scanner扫描
- 优先选择官方认证的Skill
- 敏感操作需要人工确认
